Method Details

~ (Function) JSUtils.overload(signatures1toN, functionToExecute1)

Overload functions be defining a set of signatures for a function so that function gets executed if the current call's arguments match one of the signatures.

Examples:

Simple overload example 

f = JSUtils.overload(
    [Number, String]                # \
    (a, b) ->                       # |> block
        return a + parseInt(b, 10)  # /

    [String, Number]                # \
    (a, b) ->                       # |> block
        return parseInt(a, 10) + b  # /

    [Boolean, String]   # \
    [String, Boolean]   # |
    [String, Number]    # |>  block
    (x, y) ->           # |
        return x + y    # /
)

Parameters:

  • arguments... ( Blocks ) — For what a block can be see the example and the overload section.

Overloads:

~ (Function) JSUtils.overload(signatures1toN, functionToExecute1)

Defines one function for N different signatures. So let's call such a block a list of signatures belonging to one function: Any number of blocks can be passed to JSUtils.overload().

Parameters:

  • signatures1toN... ( Array of Array of class ) — Signatures. Any number of arrays (of classes).
  • functionToExecute1 ( Function ) — Function body for the previously defined signatures.

Returns:

  • ( Function ) — The overloaded function

~ (Boolean) isSubclass(sub, sup)

Defines what is considered a subclass. Set the body of JSUtils.isSubclass to return sub == sup to disable support for subclass checking

Parameters:

  • sub ( Class ) — potential subclass
  • sup ( Class ) — potential superclass

Returns:

  • ( Boolean ) — If sub is a subclass of sup
