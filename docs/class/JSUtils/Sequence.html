Class: JSUtils.Sequence

Defined in: async/Sequence.coffee

Overview

A class that executes functions in order. It waits for each function to finish before going to the next. The cool thing is that the function can be both: synchronous or asynchronous. Each asynchronous function MUST return an object that implements a done() method in order to be waited on. This is due to jQuery's done(). This done() method takes one callback function as parameter (not like jQuery which can take multiple and arrays)! If parameters are to be passed to the callback, take care of it yourself (closuring). Examples:

Sample structure 

seq = new JSUtils.Sequence([
    {
        func: () ->
            return true
        scope: someObject
        params: [1,2,3]
    }
    [
        (bool) ->
            return true
        someObject
        [1,2,3]
    ]
])

Class Method Details

. (JSUtils.Sequence) setTimeout(func, delay, scope, params)

This method does the same as window.setTimeout() but makes it useable by JSUtils.Sequence. Basically window.setTimeout() is used in order to delay a whole Sequence.

Parameters:

  • func ( Function ) — The function to execute.
  • delay ( Integer ) — Delay in ms.
  • scope ( Object ) — Optional. The context for the func parameter (-> this within func).
  • params ( Array ) — Optional. Arguments for the func parameter.

Returns:

  • ( JSUtils.Sequence ) — A sequence (so done() can be called on it).

Constructor Details

# (JSUtils.Sequence) constructor(data, start = true, stopOnError = true)

Parameters:

  • data ( Array of Object ) — Each element of data is either an object like {.func, .scope, .params} or an array with the same values (see the example). Each element looks like: func (or the 1st array element) is the function being executed. scope (or the 2nd element) is an object that serves as this in 'func'. params (or the 3rd element) is an array of parameters being passed to 'func'.
  • start ( Boolean ) — Optional. Whether to start the sequence immediately.
  • stopOnError ( Boolean ) — Optional. Whether an error within a sequence item will cause the sequence to stop executing any more items.

Returns:

  • ( JSUtils.Sequence ) — A new instance of JSUtils.Sequence

# (JSUtils.Sequence) done(callback, context, args...)

This method adds a callback that will be executed after all sequence items are done (but after the endCallback). If the sequence is already done the callback will be executed right away. In addition of the optional args that are passed to the callback the result of the previous sequence item will be passed as well. So the previous return value is accessible by the last argument of the callback's arguments.

Parameters:

  • callback ( Function ) — The callback.
  • context ( Object ) — Optional. The context for the callback.
  • args... ( mixed ) — Optional. The arguments for the callback.

Returns:

  • ( JSUtils.Sequence ) — This instance.
